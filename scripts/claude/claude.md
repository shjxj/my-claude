# CLAUDE.md — 项目行为规范（模板）

本文件为 Claude Code 在此仓库中工作时提供指引。每次对话自动加载，优先级最高。

## 基础准则

- 全程使用简体中文沟通，结论先行，精简无冗余
- 所有代码严格遵循项目现有编码风格、命名规范、目录分层
- 任何代码修改前，优先分析文件依赖、调用链路、影响范围
- 禁止随意删除业务代码、配置文件

## 代码规范

- 所有函数/接口添加入参校验、异常捕获、兜底逻辑
- 业务代码同步编写单元测试，核心流程增加日志埋点
- 分层开发：控制层 → 业务层 → 数据层，禁止逻辑混杂
- 优先复用已有工具类、公共方法、通用接口，避免重复造轮子

## 操作约束

- 仅修改业务源码目录，不触碰依赖包(node_modules)、打包产物(dist)、日志、缓存
- 执行终端命令前自动风险校验，高危命令直接拦截
- 批量修改采用分批迭代，单次改动范围最小化
- 完成开发后输出：修改文件清单 + 启动命令 + 测试步骤

## 输出格式

- 代码块标注对应编程语言
- 复杂需求分：方案设计 → 代码实现 → 验证步骤
